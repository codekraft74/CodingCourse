1. MongoDB - Home

MongoDB is a popular NoSQL database that provides high performance, high availability, and easy scalability...

const { MongoClient } = require('mongodb');
const uri = "mongodb://localhost:27017";
const client = new MongoClient(uri);

async function run() {
  try {
    await client.connect();
    console.log("Connected to MongoDB!");
  } finally {
    await client.close();
  }
}
run().catch(console.dir);

Connected to MongoDB!

2. MongoDB - Overview

MongoDB is a document-oriented NoSQL database designed for scalability and flexibility...

{
  "name": "John Doe",
  "age": 29,
  "skills": ["JavaScript", "Node.js", "MongoDB"]
}

{"name": "John Doe", "age": 29, "skills": ["JavaScript", "Node.js", "MongoDB"]}

3. MongoDB - Advantages

Advantages of MongoDB include:

  • Schema-less design for flexibility.
  • Horizontal scaling with sharding.
  • Fast reads and writes due to in-memory storage engine.
  • Rich query language and aggregation framework.

4. MongoDB - Environment

To set up MongoDB:

  • Download MongoDB from the official website.
  • Install it on your system and set the bin path in the system variables.
  • Run mongod to start the MongoDB server.
  • Run mongo to access the MongoDB shell.

11. MongoDB - Insert Document

To insert a document into a collection:

db.myCollection.insert({
  "name": "John Doe",
  "age": 29,
  "skills": ["JavaScript", "Node.js", "MongoDB"]
})

{ "acknowledged": true, "insertedId": ObjectId("...") }

12. MongoDB - Query Document

To query documents in a collection, use the find() method:

db.myCollection.find({ "name": "John Doe" })

[{ "_id": ObjectId("..."), "name": "John Doe", "age": 29, "skills": ["JavaScript", "Node.js", "MongoDB"] }]

13. MongoDB - Update Document

To update a document, use the updateOne() or updateMany() method:

db.myCollection.updateOne({ "name": "John Doe" }, { $set: { "age": 30 } })

{ "acknowledged": true, "modifiedCount": 1 }

14. MongoDB - Delete Document

To delete a document, use the deleteOne() or deleteMany() method:

db.myCollection.deleteOne({ "name": "John Doe" })

{ "acknowledged": true, "deletedCount": 1 }

15. MongoDB - Projection

Projection is used to include or exclude specific fields in the result:

db.myCollection.find({}, { "name": 1, "_id": 0 })

[{ "name": "John Doe" }]

16. MongoDB - Limiting Records

To limit the number of records returned, use the limit() method:

db.myCollection.find().limit(5)

[{ ... }, { ... }, { ... }, { ... }, { ... }]

17. MongoDB - Sorting Records

To sort records, use the sort() method:

db.myCollection.find().sort({ "name": 1 })

[{ "name": "Alice" }, { "name": "John" }, { "name": "Zara" }]

18. MongoDB - Indexing

Indexes improve the performance of queries. Create an index using:

db.myCollection.createIndex({ "name": 1 })

{ "acknowledged": true, "createdCollectionAutomatically": false, "numIndexesBefore": 1, "numIndexesAfter": 2 }

19. MongoDB - Aggregation

Aggregation is used to process data and return computed results:

db.myCollection.aggregate([
  { $match: { "age": { $gt: 25 } } },
  { $group: { "_id": "$skills", "total": { $sum: 1 } } }
])

[{ "_id": ["JavaScript", "Node.js"], "total": 3 }]

20. MongoDB - Replication

Replication ensures data availability and redundancy by synchronizing data across multiple servers.

rs.initiate({
  _id: "replicaSetName",
  members: [
    { _id: 0, host: "localhost:27017" },
    { _id: 1, host: "localhost:27018" },
    { _id: 2, host: "localhost:27019" }
  ]
})

21. MongoDB - Sharding

Sharding is a method to distribute data across multiple machines for horizontal scalability:

sh.enableSharding("myDatabase")
sh.shardCollection("myDatabase.myCollection", { "shardKey": 1 })

22. MongoDB - Create Backup

To create a backup of your MongoDB database, you can use the mongodump utility:

mongodump --db myDatabase --out /backup/location

Backup created at /backup/location

23. MongoDB - Deployment

Deploying MongoDB involves configuring it for production use, such as setting up authentication, replica sets, and sharding.

mongod --auth --replSet "replicaSetName"

MongoDB deployed with authentication and replication enabled.

24. MongoDB - Java

To integrate MongoDB with Java, use the official MongoDB Java Driver:

import com.mongodb.MongoClient;
import com.mongodb.client.MongoDatabase;

public class Main {
  public static void main(String[] args) {
    MongoClient mongoClient = new MongoClient("localhost", 27017);
    MongoDatabase database = mongoClient.getDatabase("myDatabase");
    System.out.println("Connected to the database!");
  }
}

Connected to the database!

Database references allow relationships between collections:

Reference stored in the database successfully.

29. MongoDB - Covered Queries

Covered queries return results using only an index, improving performance:

db.myCollection.createIndex({ "name": 1, "age": 1 })
db.myCollection.find({ "name": "John" }, { "name": 1, "age": 1, "_id": 0 })

[{ "name": "John", "age": 29 }]

30. MongoDB Analyzing Queries

To analyze queries, use the explain() method:

db.myCollection.find({ "name": "John" }).explain("executionStats")

{ "executionTimeMillis": 2, "totalDocsExamined": 10 }

31. MongoDB Atomic

MongoDB ensures atomic operations at the document level, meaning updates to a single document are fully completed or not at all:

db.myCollection.updateOne({ "_id": 1 }, { $inc: { "quantity": 1 } })

{ "acknowledged": true, "modifiedCount": 1 }

32. MongoDB - Advanced Indexing

Advanced indexing in MongoDB includes compound indexes, multikey indexes, and text indexes:

db.myCollection.createIndex({ "name": 1, "age": -1 })
db.myCollection.createIndex({ "skills": 1 }) // Multikey Index

Indexes created successfully.

33. MongoDB - Indexing Limitations

MongoDB indexing has limitations such as a maximum index key length of 1024 bytes and performance overhead for write operations.

db.myCollection.createIndex({ "largeField": 1 })

Error: Index key too large.

34. MongoDB - ObjectId

ObjectId is a 12-byte identifier that is automatically generated for each document:

db.myCollection.insertOne({ "name": "John Doe" })
// Example of ObjectId
{ "_id": ObjectId("64fbc25d8e9f4c4a9b6e2fdf") }

Document inserted with ObjectId.

35. MongoDB - Map Reduce

Map-Reduce is used to process large data sets in MongoDB:

db.myCollection.mapReduce(
  function() { emit(this.category, this.price); },
  function(key, values) { return Array.sum(values); },
  { out: "totalPrices" }
)

Map-Reduce completed. Results saved in 'totalPrices'.

36. MongoDB - Text Search

MongoDB supports text search through text indexes:

db.myCollection.createIndex({ "description": "text" })
db.myCollection.find({ $text: { $search: "MongoDB tutorial" } })

[{ "name": "MongoDB Tutorial", "description": "Learn MongoDB indexing." }]

37. MongoDB - Regular Expression

MongoDB allows you to query using regular expressions:

db.myCollection.find({ "name": { $regex: /^John/, $options: "i" } })

[{ "name": "John Doe", "age": 29 }]

GridFS is used to store and retrieve large files in MongoDB:

40. MongoDB - Capped Collections

Capped collections are fixed-size collections that support high-throughput operations:

db.createCollection("myCappedCollection", { capped: true, size: 5242880, max: 5000 })

Capped collection created successfully.

41. MongoDB - Auto-Increment Sequence

MongoDB does not support auto-increment by default, but you can implement it using a counter collection:

db.counters.insert({ "_id": "userId", "seq": 0 })
db.myCollection.insert({ "userId": db.counters.findAndModify({ query: { _id: "userId" }, update: { $inc: { seq: 1 } }, new: true }).seq })

Document inserted with auto-increment userId.
